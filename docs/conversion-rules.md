# 변환 규칙

Obsidian 마크다운 → Slack mrkdwn 변환 규칙 전체 목록.

## 변환 테이블

| # | Obsidian | Slack mrkdwn | 설명 |
|---|----------|-------------|------|
| 1 | `**bold**` | `*bold*` | 별표 2개 → 1개 |
| 2 | `*italic*` | `_italic_` | 별표 → 언더스코어 |
| 3 | `***bold italic***` | `*_bold italic_*` | 볼드+이탤릭 분리 |
| 4 | `~~strike~~` | `~strike~` | 틸드 2개 → 1개 |
| 5 | `` `code` `` | `` `code` `` | 변환 없음 (유지) |
| 6 | ` ```lang\ncode``` ` | ` ```\ncode``` ` | 언어 태그 제거 |
| 7 | `# Heading` ~ `###### Heading` | `*Heading*` | 헤딩 → 볼드 |
| 8 | `[text](url)` | `<url\|text>` | 마크다운 링크 → Slack 링크 |
| 9 | `![alt](url)` | `<url\|alt>` | 이미지 → Slack 링크 |
| 10 | `[[Note]]` | `Note` | 위키링크 → 일반 텍스트 |
| 11 | `[[Note\|Alias]]` | `Alias` | alias 텍스트만 유지 |
| 12 | `![[embed]]` | `embed` | 임베드 → 일반 텍스트 |
| 13 | `> [!type] title` | `> TYPE: title` | callout → 인용문 |
| 14 | `> quote` | `> quote` | 변환 없음 (유지) |
| 15 | `- [ ] task` | `☐ task` | 미완료 체크박스 |
| 16 | `- [x] task` | `☑ task` | 완료 체크박스 |
| 17 | `==highlight==` | `*highlight*` | 하이라이트 → 볼드 |
| 18 | `$math$` | `` `math` `` | 인라인 수학식 → 인라인 코드 |
| 19 | `$$math$$` | ` ```math``` ` | 블록 수학식 → 코드블록 |
| 20 | `%% comment %%` | (제거됨) | Obsidian 주석 삭제 |
| 21 | `[^1]` + 각주 정의 | (인라인 삽입) | 각주를 본문에 병합 |
| 22 | 테이블 | 행별 텍스트 | `헤더: 값 \| 헤더: 값` 형태 |
| 23 | `---` | `───────` | 유니코드 구분선 |

## 변환 순서 (충돌 방지)

변환 순서가 중요한 이유: `*`가 볼드(`**`)와 이탤릭(`*`) 모두에 사용되므로, 볼드를 먼저 처리해야 이탤릭과 충돌하지 않음.

```
1.  코드블록/인라인 코드 → 플레이스홀더 (보호)
2.  주석 제거 (%% %%)
3.  블록 수학식 → 코드블록
4.  인라인 수학식 → 인라인 코드
5.  callout → 인용문
6.  각주 인라인 병합
7.  테이블 → 텍스트
8.  볼드+이탤릭 (***→*_..._*)
9.  볼드 (**→*)
10. 이탤릭 (*→_)
11. 취소선 (~~→~)
12. 하이라이트 (==→*)
13. 이미지 → Slack 링크
14. 마크다운 링크 → Slack 링크
15. 임베드 → 텍스트
16. 위키 링크 → 텍스트
17. 헤딩 → 볼드
18. 체크박스 → 이모지
19. 구분선 → 유니코드
20. 코드블록 플레이스홀더 복원
```

## 미지원/제한 사항

- Obsidian 플러그인 전용 문법 (Dataview 쿼리 등)은 그대로 유지됨
- 중첩된 마크다운 (예: 볼드 안의 이탤릭)은 대부분 처리되나, 복잡한 중첩은 예외 가능
- 테이블의 정렬(`:---:`) 정보는 변환 시 무시됨
